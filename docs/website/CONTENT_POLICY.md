# Content Policy for Public Website

## Purpose
Define mandatory rules for copy, visuals, and claims on the public FIELDexa website.

## Mandatory Rules
1. Use only approved assets listed in COPYRIGHT_LICENSE_REGISTER.md.
2. Publish only claims present in CLAIMS_MATRIX.md.
3. Label upcoming capabilities as In Rollout where applicable.
4. Keep legal, privacy, and cookie content aligned with internal policies.
5. Avoid confidential implementation details not intended for public visibility.

## Prohibited Content
1. Copyrighted third-party media without explicit rights.
2. Unverified customer outcomes or benchmark claims.
3. False or guaranteed compliance claims.
4. Unauthorized use of partner names, logos, or testimonials.
5. Leaked internal credentials, endpoints, or infrastructure details.

## Copy Style Rules
1. Use enterprise, precise, and trustworthy language.
2. Prefer capability and workflow statements over hype.
3. Distinguish clearly between Live Now and In Rollout messaging.
4. Keep farmer livelihood impact language responsible and non-exaggerated.

## Review Workflow
1. Draft copy by implementation team.
2. Evidence check against CLAIMS_MATRIX.md.
3. Legal-safe language check against this policy.
4. Final leadership approval before production publish.
